STEMgraphs Model

I'm a physicist, of course I want to build a model

So let me explain my model

What I disagree with

"You sometimes have to run before you learn to walk"
â€” Some Teachers

What I see as my job

make efficient use of the students limited time

make them productive

make them competent

make them able to help themself

Students and Employees are paying with their time

What I see as a journey

Everybody starts at a different point

But I want comparable outcomes

What I heard from a co-worker

Just play THPS

Everyone has differnet stats

Reaching 100% with different characters require different paths

What I mean when I say learning

Learning is an active process

It describes a trajectory

The substrate is the skillspace

There are limited valid transitions

Thus forming distinct skilltrees

What I mean by Skilltree

I wanted to become a Master Shape Shifter

Not really, but in an RPG

There were certain steps to take

I didn't know which ones

Googled and found a skilltree

What I want others to see

Think of any given task in your job

Can you write it down in atomic steps

Doesn't every step have a pre-requisite

Sort of a dependency upon a different atomic step

The ability to perform these steps is competence

I tried generating a list of these atomic steps

Ran out of paper

STEMgraphs State of Affairs

I started a few weeks ago

Why would I write them on paper

How do I store them

It's a graph

Use a graph database

Neo4j

arrange relationships of skills

Nodes - atomic challenges

Edges - dependencies

Subgraphs can be disjuncted

Will get joined later

What I use to store the challenges

I screw up a lot of times

Storing in the DB isn't convenient

Storing it in Git makes perfect sense

Collaboration, Issues, Pull Requests, History, etc

What I use to display the challenges

Neo4j comes with a browser

It's not easy to use without knowing the cypher

Build a custom UI

What if I want to change something in the DB structure

Write a Use-Case-Driven API

Implement API in FastAPI

What I use to access the challenges

Differnet people like reading different formats

I don't want to prepare all formats by hand

Started to write a DSL in LaTeX - didn't work too well

Switched to defining a JSON Schema

What I use to generate these documents

A tag is created for each release

Parser & Generator written in Python & running in Github Actions

HTML is served by Github-Pages

.md is served by Github README

.pdf is served by Github Release artifacts
